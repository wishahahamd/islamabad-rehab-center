1. Introduction & Project Background

Rehabilitation and behavioral healthcare facilities operate in a highly sensitive domain where patient privacy, data integrity, and strict clinical workflows are paramount. Currently, many rehabilitation centers in Pakistan rely on fragmented manual record-keeping systems or non-permissioned digital sheets. This lack of structure leads to administrative errors, compliance risks, and communication gaps between clinicians, case managers, and patients.

The proposed project, Islamabad Rehab Center (IRC) Portal, is a secure, lightweight, and dynamic web application designed to centralize and automate clinical administrative workflows. The portal provides dedicated operational workspaces for three key user classes: administrative officers, clinical directors/doctors, and recovering patients.

1.1 Project Purpose

The main purpose of the portal is to ensure that healthcare records—including initial patient intake documentation, daily medical detoxification progress, psychological therapy assessments, and clinician feedback logs—are securely processed and accessible under a strict **Role-Based Access Control (RBAC)** security model. By moving operations online, the facility can eliminate paper trail losses and provide families with secure updates on patient recovery.

2. Problem Statement

Traditional offline medical workflows at substance rehabilitation clinics encounter several operational challenges:

  1. Data Privacy Risks: Patient diagnosis notes, detoxification medication regimens, and therapy logs are highly confidential. Paper-based filing runs a high risk of unauthorized exposure.
  2. Communication Overhead: Outpatients and family sponsors lack direct digital access to monitor recovery stages, check appointment intervals, or securely signal emotional triggers to clinicians.
  3. Static Software Rigidity: Existing commercial management systems use hardcoded configuration files. Adding a new script page, registering custom security roles, or changing menu navigation requires code alteration by external developers.
  4. Vulnerabilities to Exploitation: Standard raw SQL query structures are prone to SQL injection inputs and Cross-Site Scripting (XSS) actions inside clinic logs.

3. Proposed Solution & Key Features

To address these issues, the Islamabad Rehab Center (IRC) Portal employs a 100% database-driven structure where security configurations, user permissions, active pages, and sidebar menus are resolved dynamically in MySQL.

3.1 Core System Modules

  • 100% Dynamic RBAC Engine: All sidebar links and script execution paths are validated against a role_access table mapping. If a user's role is not authorized for a requested page ID, execution halts immediately and renders a 403 Forbidden template.
  • Clinician Workspace: Allows doctors, clinical directors, and counselors to register new patients, schedule therapy logs, write diagnostic assessment notes, track caseload allocations, and review analytics.
  • Patient Self-Service Portal: Enables patients to view their current recovery phase (Intake, Detox, Rehab, Outpatient, Discharged), log daily mood/trigger status, communicate issues via support tickets, and export their clinical recovery PDF.
  • Admin Branding Controller: Allows super administrators to update system logos, center names, and footer details directly from a web configuration widget without modifying PHP source code.

4.1 Feasibility Evaluation

  • Technical: Core PHP and MySQL are highly mature, stable, and well-documented. Running the application under XAMPP ensures straightforward deployment.
  • Operational: Distinct layouts accommodate the specific daily workflows of intake counselors, therapists, and administrative supervisors.
  • Economic: Utilizing free open-source software structures eliminates licensing expenditures for the rehabilitation center.
