Project Vision & Positive Vibe

Empowering Recovery: Features a positive, clear orange visual layout symbolizing energy, warmth, and healing.
Daily Reflective Logs: Encourages patients to log mood shifts and triggers, fostering self-awareness and active participation in their recovery.

Healing Environment

Our platform bridges communication gaps, providing patients with a transparent recovery timeline and direct, secure access to counselors.

Detailed Project Analysis

Security Vector

Evaluated against vulnerability models. Standard prepared inputs (PDO) mitigate 100% of SQL-Injection risks in patient directories.

Workflow Automation

Eliminates clinical paper logs. Automates caseload routing from counselor assignment down to medication logs and aftercare.

System Optimization

Dynamic memory allocation via database queries avoids heavy hardcoded arrays, ensuring faster loading over local hosts.

User Adoption Index

Three tailored portal interfaces accommodate different workflows, resulting in reduced training requirements for clinical personnel.

Proposed Solution

Centralized Data Hub: All settings, roles, page URLs, and user accounts are database-driven.
Workspace Separation: Segregated views tailored for Administrators, Clinicians, and Patients.
Secure Execution: Uses PHP Data Objects (PDO) parameterized inputs to block SQL Injection.

100% Dynamic Engine

Any page additions, role changes, or branding settings update dynamically from the DB, with zero source code modifications needed.

System Architecture

3-Tier Structure: Separates presentation, business logic, and database persistence layers.
header.php Guard: Gatekeeps page execution by checking session variables and page URL permissions.

Admin Portal Operations

Dynamic Page Mapping: Create new page indexes and configure checkboxes to assign permissions.
Role Manager: Edit roles, generate custom system role keys, and protect system accounts.
Branding Customizer: Set center title and footer parameters directly from the dashboard.
Safe Role Migrations

If a custom role containing active users is deleted, the system automatically migrates all affected users to a suspended state before deleting the role, preventing orphaned records.

Clinician Portal Operations

Patient Intake Registry

Complete initial intakes, record symptoms, assign doctor-to-patient relations, and log recovery status.

Treatment Goal Planner

Formulate detailed medical detoxification phases, psychological counseling schedules, and aftercare plans.

Therapy Session Logger

Log individual/group counseling sessions, record progress reviews, and track patient progress.

Support Tickets Workspace

Track and respond to patient inquiries, input resolutions, and mark tickets resolved.

Patient Portal Operations

Daily Mood Logger: Track mood score (1-5) and log reflections, allowing doctors to view recovery trends.
Treatment Plan Monitor: Direct access to active detox goals and therapy schedules.
Confidential Reports: View session logs history and download Patient Recovery Reports.
Support Ticket Submission

Patients can submit assistance requests directly to their assigned clinician, ensuring continuous out-of-clinic communication.

Relational Database Design

Core Settings: system_settings, sys_roles, sys_pages, role_access.
User Accounts: users table storing hashed credentials and active status.
Clinical Logs: patients, therapy_sessions, treatment_plans.
Patient Telemetry: patient_daily_logs, support_requests.
Relational Mappings

All transactional operations (therapy sessions, daily logs, support requests) are mapped to patient records and users using foreign keys to ensure data integrity.

Testing & Verification

Automated Test Suite: Created test_suite.php checking roles, auth gates, and access controls.
Caseload Isolation: Evaluated database queries to ensure patients cannot access other records.
Output Sanitization: Tested forms to confirm XSS scripts are blocked using htmlspecialchars.
